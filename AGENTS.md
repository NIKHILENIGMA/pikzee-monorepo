# Project Agent Instructions

This repository is public-safe. Do not add private architecture notes, roadmap details, vendor credentials, data models, or operational workflows to this file.

## Development Guidelines

- Follow the existing project structure and coding patterns.
- Keep changes scoped to the requested feature or fix.
- Prefer shared utilities and components already present in the repository.
- Run the relevant lint, typecheck, and tests before finishing when available.
- Do not commit generated files, local environment files, credentials, or private documentation.

## Private Notes

Detailed product architecture and data modelling notes should stay in `AGENTS.private.md`, which is intentionally ignored by Git.
